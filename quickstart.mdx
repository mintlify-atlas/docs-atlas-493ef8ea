---
title: "Quickstart"
description: "Get VITACORE running locally in minutes."
---

This guide walks you through cloning the repository, configuring your environment, and opening VITACORE in a browser for the first time.

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/msgiraldo01-wq/VITACORE.git
    cd VITACORE
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    <Tabs>
      <Tab title="Linux / macOS">
        ```bash
        python3 -m venv venv
        source venv/bin/activate
        ```
      </Tab>
      <Tab title="Windows">
        ```bash
        python -m venv venv
        venv\Scripts\activate
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This installs Flask, the Supabase Python client, and python-dotenv.
  </Step>

  <Step title="Create a .env file">
    Create a `.env` file in the project root with your Supabase credentials and a Flask secret key:

    ```env
    FLASK_SECRET_KEY=your-secret-key-here
    SUPABASE_URL=https://your-project-ref.supabase.co
    SUPABASE_ANON_KEY=your-anon-key
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
    ```

    You can find `SUPABASE_URL`, `SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_ROLE_KEY` in your Supabase project under **Settings → API**.

    <Warning>
      Never commit `.env` to version control. Add it to `.gitignore` before your first commit.
    </Warning>
  </Step>

  <Step title="Run the application">
    ```bash
    python app.py
    ```

    Flask starts in debug mode on port `5000`. You should see output similar to:

    ```
     * Running on http://127.0.0.1:5000
     * Debug mode: on
    ```
  </Step>

  <Step title="Open VITACORE and log in">
    Open `http://localhost:5000` in your browser. You are redirected to `/auth/login`.

    The login flow works as follows:

    1. Enter your **username** and **password**.
    2. VITACORE looks up the `profiles` table in Supabase to resolve your username to an email address.
    3. It then calls Supabase Auth's `sign_in_with_password` with that email and the supplied password.
    4. On success, your profile and role are stored in the Flask session and you are redirected to the dashboard.

    <Note>
      User accounts must exist in both `auth.users` (managed by Supabase Auth) and the `profiles` table. See [Database setup](/setup/database) for details.
    </Note>
  </Step>
</Steps>

<Check>
  You're all set. VITACORE is running locally and you can log in to the dashboard. Explore the [User Guide](/guide/authentication/login) to learn about patients, clinical records, and role-based access.
</Check>
